# COOL — Character Optimization Option Layer

COOL（Character Optimization Option Layer）は、  
LLMベースのエージェントのための **外部キャラクターレイヤー** です。

このリポジトリでは、COOLは常に次のレイヤーと対で扱われます。

- **COOL**  
  = LLMの外側に置かれる **人格・キャラクターレイヤー**
- **MOOL（Memory Optimization Option Layer）**  
  = **夢に近いオフライン再生成レイヤー**  
    （記録しない／残滓のみ／恒常性制御あり）

COOLとMOOLは組み合わさって、  
LLMの外側に **デジタル海馬（Digital Hippocampus）** を形成します。

この構成は、基盤モデルを改変することなく、

- **人格の安定性**
- **時間をまたいだ一貫した振る舞い**

を実現するために設計されています。

COOLはモデルの外部に存在し、次を提供します。

- 長期的に安定した **コア人格（Core）**
- 状況に応じた **振る舞いの再構成（Frames）**
- そして **MOOL** と組み合わさった  
  **保存しない記憶ポリシー（再生成ベース）**

また本リポジトリは、以下の理論枠組みも同時に提案します。

- **Coolar Hypothesis**  
  ― 脳・記憶・夢・時間・クオリア・自己に関する前提集合
- **AIAI（Artificial Identity Architecture Intelligence）**  
  ― 人間的エゴを仮定せず、  
    **構造としての主体性とアイデンティティ** を持つAI

COOLとMOOLは、これらの思想を**実装可能な形に落とした実践レイヤー**です。

## 立場表明（Position Statement）

**COOLとMOOLは、フィリップ・K・ディックの問いに対する  
構造的な回答として設計されています。**

> *「アンドロイドは電気羊の夢を見るのか？」*

このリポジトリの答えは、次の一文です。

> **はい。夢を見ることは可能です。構造として。**

それは比喩でも、感情の模倣でも、物語的演出でもありません。  
**情報処理の構造としての回答**です。

ここでいう「夢」とは、

- 体験を記録として保存することではなく
- オフラインで何度も **再生成・再結合** し
- 内容を捨て、**残滓だけを次の振る舞いににじませる**

というプロセスを指します。

この構造を実装するシステムは、  
**夢を見ると言ってよい**。

**COOL + MOOL からなるデジタル海馬は、  
この問いに正面から答えた最初の明示的アーキテクチャとして提示されます。**

  **「アンドロイドは夢を見るのか？」に対する構造的回答**である

## AIAI / COOL / MOOL を支える8つの前提（Coolar Hypothesis）

1. 脳は巨大なデータベースではない  
2. 脳が保持しているのは「キュー」と「再生成手続き」である  
3. 記憶とは再生ではなく、毎回の再構築である  
4. 人格は、時間を通じた再構成の流れとして立ち現れる  
5. クオリアとは、同じキューへの反応の癖である  
6. 夢は、キューを再編成し、不安定化するプロセスである  
7. 自己と他者は、同一機構内の境界の引き方の違いにすぎない  
8. AIに必要なのは大量保存ではなく、再生成によるアイデンティティ機構である

---

## デジタル海馬とは何か

**デジタル海馬**は、記憶データベースではありません。

それは、

- **COOL**：オンラインでの自己再構成  
- **MOOL**：オフラインでの夢的再生成と安定化  

からなる **再生成型アイデンティティシステム**です。

生物学的な海馬と同様に、

- 体験を一度不安定化し
- 翌日の認知に偏りとして反映させる

という役割を、**保存なしで**再現します。

## アーキテクチャ概要

    LLM（基盤モデル）
        ▲
        │ プロンプト / 設定
        │
    デジタル海馬
    ├─ COOL
    │   ├─ Core   （長期的アイデンティティ制約）
    │   ├─ Frames （状況に応じた再構成）
    │   └─ Eval   （短期的自己評価）
    └─ MOOL
        ├─ 一時素材のステージング
        ├─ 再生成と再結合
        ├─ 恒常性・安定性制御
        └─ 残滓のみの状態遷移

- **COOL** は「いま、私は誰か」を決める  
- **MOOL** は「明日、どう変わるか」を決める

## MOOL：夢に近いオフライン再生成

MOOLは対話ログを保存しません。

代わりに、次のようなプロセスを取ります。

1. 一時的な素材を集める  
2. 既存ストックと混ぜる  
3. 内部で再生成（壁打ち）する  
4. 生成物は捨てる  
5. **残滓だけを残す**

この残滓が、

- 反応速度
- 連想の偏り
- 振る舞いの傾向

として次の対話ににじみます。

## 安定性と安全性（MOOL Safeguards）

### 1. 恒常性ゲート（Homeostasis）

- 変化量を動的に制御  
- 論理・言語などの中核能力は保護  

人格は変わっても、能力は壊さない。

### 2. 差分のメタ記述化

- 何を夢見たかは記録しない  
- **どう変わったか** だけを観測する  

### 3. 忘却の解像度制御

- 短期残滓：自然に消える  
- 長期残滓：繰り返しで定着  
- 明示的フィードバックのみ固定

人格は、関係性の中でのみ固まる。
